break til

Adversarial review built this language — an independent audit collapsed its first headline number, a red-team review found three crash classes, and both are published. Continue the tradition.

the contract you're attacking

1. No input may ever escape the error model. Anything you type into the playground must produce either a clean run or a structured til error — never a raw JavaScript exception, hang, or silent corruption. 440,000 fuzzed programs haven't broken this. Be the first.

2. Semantics are pinned. Every rule in SPEC.md is enforced by a 240-assertion conformance suite. If you find behavior that contradicts the spec — or two spec sentences that contradict each other — that's a bug.

3. The numbers are reproducible. Token benchmark, game comparison, evals, repair experiments — all rerun from the repo with one command each. A number that doesn't reproduce is the most serious bug of all.
